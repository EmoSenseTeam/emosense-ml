# EmoSense ML API

REST API на **FastAPI** для задач, связанных с эмоциональным состоянием и поведенческими индикаторами студентов: классический ML по табличным признакам, **multi-label** модерация текста (Jigsaw), а также сценарии на базе **Google Gemini** (чат и анализ эмоций по тексту).

Проект рассчитан на использование в **дипломной работе** и публикации на **GitHub**: описание ниже соответствует текущему коду репозитория.

---

## Возможности

| Область | Описание |
|--------|----------|
| **Поведенческий анализ** | Оценка стресса и сопутствующих показателей по числовым признакам (возраст, сон, экранное время и др.) с помощью обученной модели `scikit-learn`. |
| **Модерация текста** | Объединение **multi-label** вероятностей токсичности (TF-IDF + `OneVsRestClassifier` + логистическая регрессия) и **rule-based** детектора ненормативной лексики (RU/EN словари); единый JSON с итоговой меткой и действием. |
| **AI-чат** | Ответ ассистента через **Gemini API** (`requests`), режимы `soft` / `strict`, в ответе — краткий результат анализа эмоций по сообщению пользователя. |
| **Анализ эмоций текста** | Отдельный endpoint: эмоциональная метка и распределение оценок по заданным классам через Gemini и строгий JSON-промпт. |

**Хранение состояния:** история чата и журнал результатов анализа эмоций хранятся **в памяти процесса** (после перезапуска сервера данные теряются). Персистентная БД в проекте **не подключена**.

**Модерация — частично расширяемая логика:** в ответе API предусмотрены поля сигналов для категорий вроде сексуального контента, экстремизма и сексизма; в текущей реализации они заданы как **заглушки** (`detected: false`, нулевая уверенность). Итоговые правила `final_label` / `action` опираются на токсичность ML, мат нормативной лексики и пороги по угрозе / identity hate.

## Технологии

- Python 3  
- **FastAPI**, **Pydantic v2**, **Uvicorn**  
- **pandas**, **scikit-learn**, **joblib** — датасеты и ML-модели  
- **python-dotenv** — конфигурация из `.env`  
- **requests** — вызовы **Gemini** (без официального SDK `google-generativeai`)

## Быстрый старт

### 1. Окружение

```bash
python -m venv venv
# Windows:
venv\Scripts\activate
# Linux / macOS:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Переменные окружения

Создайте файл **`.env`** в корне проекта (файл в репозитории не коммитится). Для модулей **чат** и **анализ эмоций** через Gemini нужны:

| Переменная | Назначение |
|------------|------------|
| `GEMINI_API_KEY` | Ключ API Google AI Studio / Gemini |
| `GEMINI_MODEL` | Имя модели (по умолчанию в коде: `gemini-1.5-flash`, если переменная пустая) |
| `GEMINI_API_URL` | Базовый URL API (по умолчанию: `https://generativelanguage.googleapis.com/v1beta/models`) |

Поведенческий анализ и модерация текста **не требуют** переменных Gemini.

### 3. Артефакты ML (обязательно перед вызовом соответствующих API)

Папка `models/` в `.gitignore` — файлы моделей нужно **собрать локально**.

**Поведенческая модель** (`models/behavioral_model.pkl`):

```bash
python -m app.utils.prepare_dataset
python -m app.utils.train_model
```

**Модель модерации** (`models/moderation_model.pkl`):

```bash
python -m app.utils.prepare_moderation_dataset
python -m app.utils.train_moderation_model
```

Исходные данные должны лежать в `data/raw/` согласно скриптам (имена файлов см. в `app/utils/prepare_dataset.py` и `prepare_moderation_dataset.py`).

### 4. Запуск сервера

```bash
uvicorn app.main:app --reload
```

- Документация Swagger: `http://127.0.0.1:8000/docs`  
- Корневой health-check: `GET /`

## HTTP API (v1)

Базовый префикс: **`/api/v1`**.

| Метод и путь | Назначение |
|--------------|------------|
| `POST /api/v1/behavioral/analyze` | Поведенческий анализ по признакам (нужен `behavioral_model.pkl`). |
| `POST /api/v1/chat/message` | Чат с Gemini; режим `soft` или `strict`; в ответе — ответ модели и краткая эмоция по тексту пользователя. |
| `POST /api/v1/emotion/analyze` | Анализ эмоций текста через Gemini (полная схема с `scores`). |
| `POST /api/v1/moderation/text` | Модерация текста: ML multi-label + словари; в теле — `user_id` и `text` (нужен `moderation_model.pkl`). |

Типичные коды ответов: **422** — ошибка валидации Pydantic; **400** — некорректные данные на уровне сервиса; **503** — отсутствует модель/файлы, ошибка внешнего API или иная сбойная ситуация (текст в `detail`).

## Данные и этика

- Датасет **Jigsaw Toxic Comment Classification** используется для обучения модели модерации (классы токсичности в духе Kaggle / Perspective API).  
- Словари ненормативной лексики — отдельные текстовые списки в `data/dictionaries/`.  
- **Не публикуйте** реальные API-ключи в репозитории; используйте только локальный `.env`.

---

## Ограничения текущей версии

- Нет подключённой БД и очередей: состояние чата и история эмоций только в RAM.  
- Интеграция с LLM завязана на **Gemini** через HTTP; смена провайдера потребует доработки `app/services/ai_client.py`.  
- Классы модерации «сексуальный контент / экстремизм / сексизм» в JSON ответа присутствуют как **заглушки** до отдельной модели или правил.  
- Обучение модерации на полном корпусе может быть ресурсоёмким; при необходимости параметры обучения настраиваются в `app/utils/train_moderation_model.py`.
